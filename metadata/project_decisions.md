# Plate Recognition Project — Decisions

## 1. Общая архитектура решения

Pipeline обработки изображения:

1. Вход: папка с JPEG изображениями
2. Детекция таблички (YOLO)
3. Вырезание таблички из исходного изображения (в полном разрешении)
4. OCR (распознавание номера)
5. Постобработка номера
6. Переименование файлов
7. Формирование отчёта с confidence

---

## 2. Детекция (Stage 1)

- Модель: YOLO (Ultralytics)
- Тип: object detection
- Классы: 1 (`plate`)
- Формат bbox: обычный (axis-aligned)
- Обучение:
  - модель: yolov8n (baseline)
  - изображения уменьшаются до 640 px (imgsz=640)

### Важно:
- После детекции координаты bbox пересчитываются на исходное изображение
- Кроп таблички делается из оригинального изображения (без потери качества)

---

## 3. OCR (Stage 2)

- Базовая модель: EasyOCR (baseline)
- Язык: ограниченный набор символов

### Допустимые символы:
- цифры: 0–9
- буквы: a, b, c, d, e

### Ограничения:
- стрелка "N" игнорируется
- лишние символы удаляются на этапе постобработки

---

## 4. Формат номера

Номер имеет структуру:

- от 1 до 4 цифр
- опционально одна буква в конце (a–e)

Примеры:
- `7`
- `12`
- `123`
- `1234`
- `123a`

---

## 5. Постобработка номера

Правила:

- дополнение нулями до 4 разрядов:
  - `7` → `0007`
  - `12` → `0012`
  - `123` → `0123`

- буква сохраняется:
  - `12a` → `0012a`

- все символы приводятся к нижнему регистру

---

## 6. Переименование файлов

Формат имени:
`<номер>[_k][-!].jpg`


### Правила:

1. Базовое имя:
   - `0007.jpg`
   - `0123a.jpg`

2. Если один номер встречается несколько раз:
   - `0007_1.jpg`
   - `0007_2.jpg`

3. Если уверенность низкая:
   - `0007-!.jpg`
   - `0007_1-!.jpg`

4. Если табличка не найдена:
   - имя файла остаётся без изменений

---

## 7. Confidence (оценка уверенности)

Используется для:
- сортировки файлов
- определения, какие файлы проверять вручную

Источники confidence:
- детектор (YOLO)
- OCR

Возможный подход:
- итоговая уверенность = комбинация confidence детекции и OCR

Файлы с низкой уверенностью получают суффикс `-!`

---

## 8. Оценка качества OCR

Эталон:
- имя файла (если корректное)

Сравнение:
- распознанный номер vs номер в имени файла

Исключения:
- изображения, где имя файла не соответствует табличке

---

## 9. Исключения из оценки

Файл:
`metadata/ocr_eval_flags.csv`


Структура:
- filename
- use_for_eval (yes/no)
- comment

Такие изображения:
- используются для обучения детекции
- НЕ используются для оценки OCR

---

## 10. Ограничения системы

- работает полностью офлайн
- без использования облачных сервисов
- рассчитана на запуск на обычном ноутбуке
